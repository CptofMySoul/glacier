What is Glacier?

Glacier strives to be a simple library for using Amazon Glacier by providing an intuitive and easy to use API. Because of its license it can be included in any project.

How to install Glacier?

To install Glacier you can use easy_install or pip:

easy_install glacier

pip install glacier

You can also pull the source code and install glacier manually:

git clone git://github.com/paulengstler/glacier.git
cd glacier
python setup.py install (run as admin/root)

So, how easy to use is Glacier?

Once you get the hang of it, it will be really simple to use Glacier. This library has three built-in types ready to use:

  • Connection - The gate to connect to Amazon Glacier in the right region
  • Vault - A data storage representation to access and store your data
  • Archive - A representation of either local or remote data in Amazon's Glacier
from glacier import Connection, Vault, Archive

# establish the connection
connection = Connection("Access Key","Secret Access Key")

# get the vault "my_vault"
vault = connection.get_vault("my_vault")

# get the local file "myfile.txt"
archive = Archive("/home/paul/myfile.txt")

# get the remote archive with its cryptic id
rarchive = Archive("normallythisshouldbe138byteslong")

# delete the remote archive
vault.delete(rarchive)

# upload the file in "my_vault"
vault.upload(archive)

# be happy :)
print "Great! Now my file has the ID " + archive.id

Actually, it will be more difficult to understand how Amazon Glacier itself works rather than this library.

What are the downsides?

Glacier is pretty much untested and lacks some functionality (multipart uploads, proper error handling). However, that will change soon. Also, Glacier has no archive ID storage yet. You will have to store the archive IDs by yourself until there will be a module.
